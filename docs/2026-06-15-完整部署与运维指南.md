## 1. 整体架构

```
┌──────────────────────────────────────────────────────────────┐
│  macOS / Windows / Linux 桌面                                │
│  ┌────────────────────────────────────────┐                 │
│  │ 鱼泡语音助手.app (Tauri 2)             │                 │
│  │  - 全局快捷键录音                        │                 │
│  │  - 上传 wav 到 FUNASR_ENDPOINT/transcribe │                │
│  │  - 调用 LLM 润色(可关闭)                │                 │
│  │  - 写入文本到光标位置                    │                 │
│  └──────┬─────────────────┬──────────────┘                 │
└─────────┼─────────────────┼────────────────────────────────┘
          │ multipart POST   │ HTTPS Chat Completion
          │ /transcribe      │ /chat/completions
          ▼                 ▼
┌──────────────────┐   ┌────────────────────────────┐
│ FunASR 服务      │   │ LLM 服务(可配置)          │
│ 端口 10095       │   │ 默认: DeepSeek-V3 官方 API │
│ 模型: Paraformer- │   │ 备选: GLM-4-Flash / Qwen  │
│       large      │   │        / Ollama / vLLM    │
│ + VAD + 标点     │   │                            │
│ 部署: 本机或服务器│   │ 部署: 全部走云 API 或自建 │
└──────────────────┘   └────────────────────────────┘
```

**两条网络路径**:
1. **ASR 路径**:桌面端 → FunASR HTTP 服务(本地或局域网,延迟低)
2. **LLM 路径**:桌面端 → LLM 提供商的 `/chat/completions`(公网,延迟 1-3s)

## 4. 配置文件全集

### 4.1 `.env`(已 gitignore,本机独有)

完整内容参考当前 [.env](.env),关键变量:

```bash
# === ASR ===
ASR_ENGINE=funasr
FUNASR_ENDPOINT=http://10.254.81.32:10095
FUNASR_MODEL=iic/speech_seaco_paraformer_large_asr_nat-zh-cn-16k-common-vocab8404-pytorch
FUNASR_VAD_MODEL=iic/speech_fsmn_vad_zh-cn-16k-common-pytorch
FUNASR_PUNC_MODEL=iic/punc_ct-transformer_zh-cn-common-vocab272727-pytorch
FUNASR_DEVICE=cpu
FUNASR_QUANTIZE=0

# === LLM ===
DEEPSEEK_API_KEY=<你的 key>
DEEPSEEK_MODEL=deepseek-chat
LLM_BASE_URL=    # 留空走 DeepSeek 官方;填写则用对应 OpenAI 兼容服务

# === 行为开关 ===
POLISH_ENABLED=1   # 1/true 启用 AI 润色,0/false/no/off 关闭
```

默认请求顺序:

```bash
1. 先请求 FUNASR_ENDPOINT（默认 `http://10.254.81.32:10095`）
2. 如果远端失败，再自动回退到 `http://127.0.0.1:10095`
```

### 4.2 启动脚本环境变量(可选,会覆盖 .env)

```bash
FUNASR_HOST=0.0.0.0           # 默认 127.0.0.1,服务器部署改 0.0.0.0
FUNASR_PORT=10095
FUNASR_PYTHON=python3.9       # 指定 Python 解释器
FUNASR_VENV_DIR=.venv-funasr-py39
FUNASR_QUANTIZE=1             # 开启 INT8 量化(更快,精度略降)
FUNASR_DISABLE_VAD=0          # 1 关闭 VAD
FUNASR_DISABLE_PUNC=0         # 1 关闭标点恢复
```

## 5. 本地部署(本机单用户)

### 5.1 前置条件

- macOS 12+(推荐 13+)
- Python 3.9 ~ 3.11(Paraformer 跑 PyTorch 2.2,**不支持 3.12+**)
- Rust 1.74+
- Node.js 18+
- 麦克风权限(系统设置 → 隐私与安全性 → 麦克风)
- 辅助功能权限(系统设置 → 隐私与安全性 → 辅助功能)

### 5.2 一键启动

```bash

cd /Users/black/IdeaProjects/vibeCoding/local-voice-assistant
./scripts/start_paraformer_service.sh
# 首次会自动:建 venv → pip install funasr/modelscope/fastapi/uvicorn → 下载 1.5GB 模型

# 2. 验证
curl http://127.0.0.1:10095/health
# 期望:{"ok":true,"engine":"funasr-paraformer","model":"iic/speech_seaco_paraformer_large_asr_nat-zh-cn-16k-common-vocab8404-pytorch",...}

# 3. 配置 .env(已存在则检查)
cat .env
# 确认 FUNASR_ENDPOINT=http://10.254.81.32:10095
# 若远端不可用,桌面端会自动回退到 http://127.0.0.1:10095
# 确认 DEEPSEEK_API_KEY=<你的 key>
# 确认 LLM_BASE_URL=(留空)
# 确认 POLISH_ENABLED=1

# 4. 启动桌面端(开发模式)

# 或打开已构建的 .app
```

## 6. 远程服务器部署(供 AI 执行)

> 这一节是给高级 AI 助手看的,目标是在**远程 Linux/macOS 服务器**上把 Paraformer 服务跑起来,让局域网/公网内的多台 Mac 客户端共享。
> 服务器示例:`10.254.81.32`(用户已有的旧 ASR 服务器,现在替换为 Paraformer)。

### 6.1 决策表

| 场景 | 推荐 |
|------|------|
| 1-3 个用户同时用,内网部署 | 4 核 CPU + 8 GB 内存,**CPU 模式够用** |
| 5-10 个用户 | 8 核 + 16 GB,或 1 张入门 GPU |
| 20+ 用户 | 16 核 + 32 GB + NVIDIA GPU(走 `funasr-onnx-gpu`) |
| 想要 < 1s 端到端 | 必须 GPU + INT8 量化 |

### 6.2 Linux 服务器一键部署流程(供 AI 逐步执行)

```bash
# === 步骤 1:环境准备 ===
# 1.1 安装 Python 3.9-3.11(不支持 3.12+)
# Ubuntu/Debian:
sudo apt update
sudo apt install -y python3.9 python3.9-venv python3.9-dev build-essential cmake git

# CentOS/RHEL:
sudo yum install -y python39 python39-devel cmake git gcc

# macOS 服务器:
brew install python@3.9 cmake

# 1.2 确认 Python 版本
python3.9 --version  # 应输出 Python 3.9.x

# === 步骤 2:拉取代码 ===
sudo mkdir -p /opt/lva
sudo chown $USER:$USER /opt/lva
cd /opt/lva
git clone https://github.com/blackwait/local-voice-assistant.git .
# 或者 scp 整个项目目录:
#   scp -r local-voice-assistant/ user@server:/opt/lva/

# === 步骤 3:启动 Paraformer 服务(后台) ===
cd /opt/lva
FUNASR_HOST=0.0.0.0 \
  nohup ./scripts/start_paraformer_service.sh > /var/log/paraformer.log 2>&1 &
disown

# 首次启动会:
#   - 自动创建 .venv-funasr-py39/
#   - 自动 pip install funasr, modelscope, fastapi, uvicorn, python-multipart, torch, torchaudio
#   - 自动从 modelscope 下载 1.5 GB 模型到 ~/.cache/modelscope/
# 耗时:5-15 分钟,看服务器带宽

# === 步骤 4:验证 ===
sleep 5
curl http://127.0.0.1:10095/health
# 期望返回 ok=true, model=paraformer

# === 步骤 5:防火墙放行 ===
# Linux:
sudo ufw allow 10095/tcp
# macOS:
sudo /usr/libexec/ApplicationFirewall/socketfilterfw --add /opt/lva/.venv-funasr-py39/bin/python

# === 步骤 6:从远程测试 ===
# 在你的 Mac 上:
curl http://10.254.81.32:10095/health
# 期望同上
```

### 6.3 systemd 开机自启(Linux 推荐)

**AI 可执行命令**:

```bash
# 创建 systemd unit 文件
sudo tee /etc/systemd/system/paraformer.service > /dev/null <<'EOF'
[Unit]
Description=Paraformer-large ASR Service
After=network.target

[Service]
Type=simple
User=lva
WorkingDirectory=/opt/lva
Environment="FUNASR_HOST=0.0.0.0"
Environment="FUNASR_PORT=10095"
ExecStart=/opt/lva/.venv-funasr-py39/bin/python /opt/lva/scripts/funasr_paraformer_server.py
Restart=on-failure
RestartSec=5
StandardOutput=append:/var/log/paraformer.log
StandardError=append:/var/log/paraformer.err

[Install]
WantedBy=multi-user.target
EOF

# 创建专用用户(可选)
sudo useradd -r -s /bin/false lva
sudo chown -R lva:lva /opt/lva

# 启用并启动
sudo systemctl daemon-reload
sudo systemctl enable paraformer
sudo systemctl start paraformer
sudo systemctl status paraformer
```

### 6.4 客户端配置变更

默认 `.env` 已优先指向 `http://10.254.81.32:10095`。如果要切换到其他服务器,只需改 `FUNASR_ENDPOINT` 为新的 IP 或域名。

如果新地址不可用,桌面端仍会自动回退到本机 `http://127.0.0.1:10095`。

**或**在 app UI 的 "AI 设置 → FunASR 服务地址" 直接改,保存即可。

**桌面端代码无需任何修改**(`/transcribe` 接口契约完全兼容)。

### 6.5 服务器健康监控(可选)

```bash
# 加 cron,每 5 分钟检查一次,失败发邮件
(crontab -l 2>/dev/null; echo "*/5 * * * * ! curl -fsS http://127.0.0.1:10095/health >/dev/null && echo 'Paraformer down at $(date)' | mail -s 'ALERT' admin@company.com") | crontab -
```

或写个 `monitor.sh`:

```bash
#!/bin/bash
URL="${1:-http://127.0.0.1:10095/health}"
if ! curl -fsS --max-time 5 "$URL" > /dev/null; then
  echo "[$(date)] DOWN: $URL"
  # 重启服务
  systemctl restart paraformer
fi
```

### 6.6 模型升级(后续)

FunASR 会持续更新模型。如要升级:

```bash
cd /opt/lva
# 改 .env 的 FUNASR_MODEL / FUNASR_VAD_MODEL / FUNASR_PUNC_MODEL
sudo systemctl restart paraformer
# 新模型会自动下载到 ~/.cache/modelscope/hub/models/iic/...
```

### 6.7 AI 部署时的回滚方案

如果 Paraformer 出问题想切回旧 SenseVoice:

```bash
# 停 Paraformer
sudo systemctl stop paraformer

# 改 .env
FUNASR_ENDPOINT=http://10.254.81.32:10095  # 旧 SenseVoice 地址
FUNASR_MODEL=iic/SenseVoiceSmall

# 桌面端重启即可,会自动用回旧配置
```

## 8. 故障排查

### 8.1 Paraformer 启动失败

| 现象 | 原因 | 解决 |
|------|------|------|
| `ModuleNotFoundError: funasr` | pip 没装 | 看 `/tmp/paraformer.log`;手动 `.venv-funasr-py39/bin/pip install -r requirements-funasr.txt` |
| `python 3.12 not supported` | Python 太新 | 装 3.9-3.11,设 `FUNASR_PYTHON=python3.11` |
| 首次启动后 1.5GB 模型下载卡住 | 网络 | 重试或手动 `modelscope download --model iic/speech_seaco_paraformer_large_asr_nat-zh-cn-16k-common-vocab8404-pytorch` |
| 10095 端口占用 | 旧服务在跑 | `lsof -iTCP:10095` → `kill` PID |

### 8.2 转写为空 / 乱码

| 现象 | 排查 |
|------|------|
| 转写返回 `{"text": ""}` | 看服务日志,大概率是 wav 格式不对(必须是 16kHz/16bit/mono) |
| 桌面端报 "transcribe failed" | 浏览器 F12 看具体 HTTP 状态码;500 = server 内部错误 |
| 服务 hang 住 | 大概率内存不够,加内存或降模型:换 `FUNASR_MODEL=iic/speech_paraformer_asr_nat-zh-cn-16k-common-vocab8404-pytorch`(small 版) |

### 8.3 LLM 调不通

| 现象 | 排查 |
|------|------|
| 401 Unauthorized | `DEEPSEEK_API_KEY` 过期或被 revoke |
| 404 Not Found | `DEEPSEEK_MODEL` 名字写错(必须 `deepseek-chat`,不是 `deepseek-v4-flash`) |
| 慢(> 5s) | 走了第三方中转。检查 `LLM_BASE_URL` 是否留空 |
| "无配置" 提示 | `DEEPSEEK_API_KEY` 和 `DEEPSEEK_ENDPOINT` 都为空 → 自动用 hardcoded fallback。**不要**依赖 fallback,填入真实 key |

### 8.4 桌面端无响应

```bash
# macOS
pkill -f "鱼泡语音助手"
# 删配置重置(慎用)
rm ~/Library/Application\ Support/com.black.local-voice-assistant/config.json
```

### 9.1 本对话中已暴露的凭证(必须 revoke)

| 类型 | 值 | 风险 |

| DeepSeek API key | `sk-***已打码***` | ⚠️ 出现在 `lib.rs` 硬编码 + benchmark 命令 |

| 旧 token 文件 | `/Users/black/IdeaProjects/github_token` | 已废弃,建议删除 |

**操作**:
1. [DeepSeek 控制台](https://platform.deepseek.com/api_keys) → revoke 旧 key → 生成新 → 写进 `.env`
2. [智谱控制台](https://open.bigmodel.cn/user-center/apikeys) → 同上
3. [GitHub PAT 列表](https://github.com/settings/tokens) → revoke 旧 token
4. 删除 `/Users/black/IdeaProjects/github_token` 文件
5. 改 [src-tauri/src/lib.rs#L25](src-tauri/src/lib.rs) 那个硬编码的 `DEFAULT_DEEPSEEK_API_KEY`,建议清空

### 9.2 长期最佳实践

- **GitHub 凭证**:用 SSH key 替代 PAT;fine-grained PAT 设 7-30 天自动过期
- **API key**:从不进 git,从不进聊天;只用 `.env` 或环境变量
